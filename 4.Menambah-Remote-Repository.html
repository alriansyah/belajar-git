Proses Berbagi :
        NK : Aku menyelesaikan sebuah commit! Sekarang persiapannya selesai, kan?
        NK : Tapi, sekarang bagaimana bisa aku membagikan file?
        GD : Git menggunakan sesuatu yang disebut remote (atau remote repository) untuk menyimpan file-file yang dibagikan.
        GD : Pengembang membagikan file dengan mengunggah ke dan mengunduh dari remote.

        Mengatur Remote :
        NK : Oke. Jadi aku perlu membuat remote?
        GD : Iya. Tapi kita tidak perlu mengkhawatirkannya sekarang karena kita dapat menggunakan sebuah layanan, seperti Github,  
            untuk membuatnya dengan mudah.
        GD : Kali ini aku sudah menyiapkan sebuah remote, jadi kita sudah siap.
        GD : Aku akan menjelaskan bagaimana cara membuat remote nanti, jadi tidak usah khawatir.

        Menambah Remote :
        GD : Kamu perlu URL remote untuk mengunggah. Ini adalah URL remote yang aku siapkan kali ini https://prog-8.com/mysite.git
        GD : Saat kamu menambahkan remote, kamu akan memerlukan nama. Biasanya, pengembang menggunakan nama origin.
        GD : Lebih spesifik, untuk mendaftarkan remote, ketik git remote add remote_name URL, seperti pada gambar di kanan.
        NK : Baik! Aku akan mencobanya!

        Lakukan/ex : git remote add remote_name URL / git remote add origin URL
